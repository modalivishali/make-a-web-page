Important Concepts

The Basics of the Web and HTML

The Web

The Web was invented around 1990's and approximately has 30 billion pages today. Whether it is a Document or an image or a video on the web they are linked or glued together through HTML .The Web is a collection of many HTML documents.When a person uses the internet through his browser , a request is placed to the server through HTTP (Hypertext Transfer Protocol) . Then the Server finds that particular HTML document and sends it to the user through the same channel. Here is a video that explains the concept more clearly.

HTML

HTML stands for Hyper text Markup Language . HTML is made up of text content (what you see ), markup (what it looks like), links to other pages and reference to other documents (like videos or images ).

HTML Markup

HTML Markups are set of commands inserted in a file for a particular display on the browser. They are made up of elements and tags. An example of basic tag is explained in this particular video . Tags are single commands placed between brackets whereas elements are components of HTML document. This video explains it in detail.

HTML Attributes

When a user while writing a HTML Document wants to refer a particular link on the web in this particular document, he uses the HTML attributes for the tag which in this case is an anchor tag. A clear example is given in this video

Inline (vs) Block Element

Both these elements are used to organise the content. But there is a difference in the usage. Inline elements can display the organised content and span the space of the content. Whereas block elements place an invisible block around the content allowing it to be displayed as a new line. Unlike inline elements, block elements occupy the width of the parent if no width is specified. Here is a video thats explains this concept.

Whitespace

While creating HTML document in order to add spaces between the lines or to create a new paragraph, there must be a specific command in the text. otherwise, all the spaces given through tab are deleted while displaying the content. The additional space is called the whitespace

Creating a Structured Document with HTML

Developer Tools

A developer tool enables a developer to view the structure of the page as interpreted by browser. While coding if there are any errors or any visual changes that are required those can be identified through developer tools. By using inspect element we can look into the details of an element.

Necessary Components of the Web

HTML, CSS and JavaScript are the important components of the web. HTML is the structure of the webpage,CSS controls the style of the page. whereas Javascript is a scripting language which allows to manupalate the DOM. DOM represents tree like structure of a page. These three components together form the Web page.

Tree-like structure

Elements can have other elements inside them which forms a structure similar to the family tree structure and hence the name. In a browser this structure comes up as a series of boxes inside other boxes and so on.

Boxes and Grid

Everything on the webpage is made up of grids or boxes. Even if the images are of irregular shapes CSS styles it in a rectangular box. Boxes are placed around every single element that is to be displayed on the page.

Tools programmer's use to write a code

Programmers use special text editing program like sublime text to code HTML document. A text editor is a program that helps to type the text in a particular format that computer understands. later, that editorial is converted in a document that other people read.

Adding CSS Style to HTML Structure

CSS

CSS is Cascading Style Sheets . It also can be interpreted as a search and replace tool. Idetify a class/tag of the element you want to find, and than what you want to do with it (or) add property values to replace it with different ones. CSS is a code written to control the style of a page.

Avoiding Repetition

While writing a Code it is very essential to avoid repetitions otherwise, programmers have to do the same thing again and again. Doing it repetitie times can lead to errors. Some errors are so small that it can become very difficult to locate them.

Summary

The process is simply divided into three parts Code , Test and refine . First programmers will write a code, than test it and finally if required make any changes. There are a few steps involved.
steps:

  1. Get the structure of the page correct. This is done by boxifying the elements. Everything on the page is put in boxes.
  2. Next, Move on to the tags and the class. If required add headings, text and other tags. Make sure opening tags have closing tags.
  3. Position the boxes on the grid correctly. Apply styles.
  4. Test the code regularly and if required make correction. Repeat the above steps until desired display is available
